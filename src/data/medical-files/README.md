# 医学知识库文件规范

## 目录结构

```
src/data/medical-files/
├── README.md                    # 本规范文档
├── {疾病名}/
│   ├── 01-诊疗指南.md           # 疾病总览
│   ├── 02-诊断标准.md           # 诊断要点
│   ├── 03-手术操作规范.md       # 手术/治疗操作
│   ├── 04-术后管理.md           # 随访、并发症处理
│   ├── 05-临床路径.md           # 标准化诊疗流程
│   └── 06-循证医学.md           # 指南来源、证据等级
```

## 文件命名规则

### 格式
```
{序号}-{模块名}.md
```

### 序号规则
| 序号 | 模块 | 说明 | 必需 |
|------|------|------|------|
| 01 | 诊疗指南 | 疾病总览、概述 | ✅ 必需 |
| 02 | 诊断标准 | 诊断要点、鉴别诊断 | ✅ 必需 |
| 03 | 手术操作规范 | 手术步骤、技巧、术式选择 | ✅ 必需 |
| 04 | 术后管理 | 随访计划、并发症处理 | ✅ 必需 |
| 05 | 临床路径 | 标准化诊疗流程 | ⚠️ 推荐 |
| 06 | 循证医学 | 指南来源、证据等级 | ⚠️ 推荐 |
| 07-99 | 补充模块 | 特殊内容 | ⚠️ 可选 |

### 命名约束
- 序号：2 位数字，范围 01-99
- 模块名：使用中文，简洁明确（2-6 字）
- 文件扩展名：`.md`
- 不允许空格、特殊字符

## 内容模板

每个文件必须包含以下章节：

```markdown
# {模块名}

## 适应症
- 本模块适用的患者类型/疾病分期

## 核心要点
- 1. ...
- 2. ...
- 3. ...

## 详细内容
### 子主题 1
...

### 子主题 2
...

## 注意事项
- 禁忌症
- 特殊情况处理

## 参考来源
- 指南名称 + 年份
- 证据等级
```

## 疾病目录命名

- 使用 `diseaseCategories.js` 中的 `name` 字段（中文）
- 示例：`腹股沟疝`、`急性阑尾炎`、`胆囊结石`

## 内容质量要求

1. **准确性**：内容必须基于权威临床指南
2. **完整性**：覆盖诊断、治疗、随访全流程
3. **实用性**：面向临床医生，提供可操作的指导
4. **规范性**：使用标准医学术语

## 扩展新疾病

1. 在 `src/data/medical-files/` 下创建疾病目录
2. 按照序号创建 01-06 文件
3. 确保内容符合上述模板
4. 验证文件名排序正确

## 注意事项

- 文件名前缀数字决定注入顺序（系统按 `sort()` 排序）
- 所有 `.md` 文件都会被自动拼接注入 AI 系统提示词
- 单个疾病知识库总字符数限制为 8000 字符
- 超出限制时系统会自动截断并记录日志
