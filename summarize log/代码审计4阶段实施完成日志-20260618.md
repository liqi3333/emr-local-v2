### A6 临床安全修复详情

**实际触发点**（审计未识别）：`PatientManager.js:375 selectPatient()`，不是 `store.reset()`

**3 个 switch 点**：
1. `PatientManager.js` — `selectPatient()` 点击切换患者时
2. `PatientManager.js` — `_switchToNextPatient()` 自动切患者时
3. `PatientManager.js` — `selectPatientByClick()` 通过病历卡片切换时

**守卫拦截链**：
- `store.setTypeData(data)` → 写入前打 `_patientId = currentPatient.id` 时间戳
- `EmrPreview._showHistory()` → 选择历史记录时校验 `_patientId` 匹配
- `EmrPreview.save()` / `regenerate()` → 保存前校验 `_patientId` 匹配

### B1+B2+B3 原子修复

审计"提醒 1"明确要求三者在同一批次修复，因为：
- 无 typeConfig（B3）时，category 写入无意义
- 无 typeConfig 时，registry content（B2）无法正确注入

| F4 | 提示词模板修复 | `promptTemplates.js` / `PromptEditor.js` | `assembleUserPrompt` 增加 `findTypeByTemplateKey` Layer2 回退（与 `assembleSystemPrompt` 对齐）|

### F1 知识库注入路径

```
_generateCore()
  → knowledge.getKnowledge(disease)
    → 读取 src/data/medical-files/{disease}/*.md
    → 按文件名排序，拼接 ≤8000 字符
  → 追加到 systemPrompt：请参考以下临床指南···
  → callAI(prompts)
```

### F3 进化数据流

```
POST /api/evolution/:templateKey
  → templateEvolution.evolveTemplate(templateKey)
    → database.getRecords() 获取≥3样本
    → 验证样本量（<3 时返回 400）
    → ai.callAI() 分析字段模式
    → 保存进化版本到 SQLite settings (key: template_evolution_v{version})
  → 返回 fieldInsights[]
```

### F4 根因

`assembleUserPrompt`（Line2 回退缺失）→ 自定义类型在预览端点返回"未知类型"。`assembleSystemPrompt` 已有 fallback 但 `assembleUserPrompt` 没有，两处不一致导致。
