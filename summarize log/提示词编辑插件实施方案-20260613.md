# AI 提示词编辑插件实施方案

**项目**：emr-local-v2（病历系统 v2）  
**日期**：2026-06-13  
**目标**：添加一套可 GUI 编辑的 AI 病历生成提示词模板系统，支持另存为自定义模板文件，保留并随时恢复默认模板，且能与未来新增的病历类型/字段自动同步。

---

## 一、需求背景

当前 `src/routes/api.js` 中，7 种病历（首次病程录、主治查房、主任查房、术前小结、术前讨论、手术记录、出院小结）的 AI 提示词全部硬编码在代码中。用户无法通过界面调整这些提示词，也无法保存多套自己的提示词方案。

本次任务新增一个"提示词编辑插件"，以独立页面 `/prompts` 的形式提供 GUI，允许用户：

1. 查看系统默认提示词模板；
2. 基于默认模板创建并编辑自定义提示词模板；
3. 将当前模板"另存为"新的模板文件；
4. 设置当前使用哪一套模板；
5. 随时恢复默认模板；
6. 当系统默认模板更新（新增标签页或字段）时，自定义模板能自动同步新增内容。

**约束条件**：

- 不改动病历主页面（`index.html` 的病历标签结构、`EmrPreview.js`、`store.js` 病历状态、`database.js` 的 `RECORD_DATA_COLUMNS` 均保持不变）。
- 不编辑 AI 对话（`ChatArea.js`）的提示词。
- 新增字段只能随默认模板更新自动同步，用户不能在编辑器里手动增删字段。
- 新增字段在自定义模板中出现的位置，必须与病历输出中的字段顺序一致。

## 二、整体架构

### 2.1 核心概念

- **默认模板**：随系统发布的只读提示词配置，位于 `src/data/defaultPrompts.json`，带有版本号。
- **自定义模板文件**：用户通过 GUI 创建/另存为的文件，存放于 `data/prompt-templates/*.json`。
- **激活模板**：当前 AI 生成实际使用的模板，通过 SQLite `settings` 表记录文件名。
- **合并模板**：生成时，用默认模板作为基准，再用自定义模板中用户编辑过的内容覆盖对应部分，缺失字段按默认模板补充。

### 3.3 数据库存储

#### settings 表

```sql
CREATE TABLE IF NOT EXISTS settings (
  key TEXT PRIMARY KEY,
  value TEXT NOT NULL,
  updated_at TEXT DEFAULT (datetime('now'))
);
```

用于存储 `active_prompt_template`（当前激活模板文件名）。

### 3.4 占位符约定

生成时替换以下占位符：

| 占位符 | 含义 |
|--------|------|

| `{{patientContext}}` | 患者基本信息 JSON |
| `{{emrContext}}` | 首次病程录内容 |
| `{{attendingContext}}` | 主治查房内容 |
| `{{preopContext}}` | 术前小结内容 |
| `{{surgeryContext}}` | 手术记录内容 |

## 四、后端 API 设计

### 4.1 模板管理 API（`src/routes/prompts.js`）

```
GET    /api/prompts/templates              列出所有模板（默认 + 自定义）
GET    /api/prompts/templates/:name        读取指定模板内容
POST   /api/prompts/templates              新建自定义模板
PUT    /api/prompts/templates/:name        保存/更新自定义模板
POST   /api/prompts/templates/:name/duplicate  另存为新的模板文件
DELETE /api/prompts/templates/:name        删除自定义模板
GET    /api/prompts/active                 获取当前激活模板文件名
POST   /api/prompts/active                 设置当前激活模板文件名
GET    /api/prompts/merged                 获取激活模板与默认合并后的最终配置
```

### 4.2 生成 API（`src/routes/api.js`）

保留现有 7 个端点：

```
POST /api/emr/generate
POST /api/attending/generate
POST /api/chief/generate
POST /api/preop/generate
POST /api/discussion/generate
POST /api/surgery/generate
POST /api/discharge/generate
POST /api/emr/generate/stream
```

内部逻辑统一改为：
1. 读取激活模板；
2. 与默认模板合并（保证字段顺序与默认一致）；
3. 替换占位符；
4. 组装 messages；
5. 调用 AI 服务。

## 六、合并算法

### 6.1 模板合并（生成时）

```
merged = deepClone(defaultTemplate)
for each key in defaultTemplate.templates:
    if customTemplate.templates[key] exists:
        merged.templates[key].rolePrompt = custom.rolePrompt || default.rolePrompt
        merged.templates[key].outputFormat = custom.outputFormat || default.outputFormat
        merged.templates[key].endingPrompt = custom.endingPrompt || default.endingPrompt
        merged.templates[key].userPrompt = custom.userPrompt || default.userPrompt
        
        # 字段按默认模板的顺序遍历
        for each fieldKey in default.templates[key].fields:
            if custom.templates[key].fields[fieldKey] exists:
                merged.templates[key].fields[fieldKey] = custom.templates[key].fields[fieldKey]
            else:
                merged.templates[key].fields[fieldKey] = default.templates[key].fields[fieldKey]
```

### 6.2 默认模板更新同步

当检测到 `defaultPrompts.json` 版本更新时：

1. 遍历默认模板的每个标签页；
2. 如果默认模板新增标签页，则整套追加到自定义模板；
3. 如果默认模板新增字段，则按默认字段顺序插入到自定义模板对应位置；
4. 已有字段的内容不覆盖用户编辑；
5. 更新自定义模板的 `defaultVersion` 为新版本号。

## 八、最终呈现结果

实施后，用户将获得：

1. **主页入口**：在病历系统主页 header 看到"✏️ 提示词"按钮。
2. **独立编辑页**：打开 `/prompts` 后看到 7 个病历类型的标签页。
3. **模板管理**：可以选择"默认模板"或自己保存的模板；支持新建、另存为、删除、设为默认。
4. **字段级编辑**：每个标签页下按病历输出顺序列出所有字段，可编辑字段显示名和说明。
5. **实时预览**：编辑后能看到最终组装出来的 system prompt。
6. **安全恢复**：随时通过"恢复此页默认"或切换回"默认模板"恢复系统原始提示词。
7. **自动同步**：系统升级默认模板后，自定义模板会自动按病历输出顺序合并新增字段，不覆盖用户已有编辑。

---

## 九、明确不做的事项

- 不修改病历主页面（`index.html` 病历标签结构、`EmrPreview.js`、病历状态、数据库表结构）。
- 不编辑 AI 聊天区域（`ChatArea.js`）的提示词。
- 不支持在编辑器里手动新增/删除字段。
- 不支持调整字段顺序。
- 新增病历标签页不会自动出现在病历主页面（只出现在提示词编辑器）。
