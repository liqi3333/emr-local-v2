## 一、项目概述

### 1.1 需求摘要

在左侧疾病栏实现管理插件，支持：
- 添加/删除/修改一级分类和二级疾病
- 修改分类的图标、颜色样式
- 新增疾病自动支持所有现有病历功能

### 1.2 核心设计决策

| 决策项 | 选择 |
|--------|------|
| 引入唯一 ID | ✅ 分类和疾病均带 UUID |
| 疾病名全局唯一 | ✅ 不允许重复 |
| 导入/导出 | ✅ JSON 格式 |
| 排序方式 | 上移/下移按钮 |
| 重命名迁移历史记录 | ✅ 事务内同步更新 `records.disease` |
| 二级疾病图标 | ❌ 不需要 |
| 颜色配置 | 1 个主色自动派生 3 字段 |

### 1.3 关键边界

> 疾病 ID 只在目录管理层使用；疾病名字符串仍是传给 AI、存入病历记录、显示在历史记录中的值。

**重命名事务逻辑**（方案 A）：
```javascript
function renameDisease(categoryId, diseaseId, newName) {
  // 1. 校验新名字全局唯一
  // 2. 在事务中：
  //    a. 更新目录中的疾病名
  //    b. UPDATE records SET disease = ? WHERE disease = ?
  // 3. 原子性：全部成功或全部回滚
}
```

## 七、使用说明

### 首次使用

1. 启动服务器：`npm run dev`
2. 打开 http://localhost:8000
3. 点击顶部 **🏥 疾病管理** 按钮
4. 在管理页面中：
   - 点击 **+ 添加** 新建分类
   - 在分类下点击 **+ 添加** 新增疾病
   - 点击 **✏️** 编辑（支持调色板 + emoji 选择器）
   - 点击 **🗑️** 删除（二次确认）
   - 点击 **▲/▼** 排序
   - 点击 **导入/导出** 备份配置
5. 回到主界面，左侧疾病树自动显示新内容
6. 点击新疾病，AI 正常生成所有病历类型

### 注意事项

- 切换标签页回来时，疾病目录会自动刷新
- 重命名疾病会同步更新所有历史病历记录中的疾病名
- 删除疾病不会删除历史病历记录，只是侧边栏不再显示
- 恢复默认会覆盖所有自定义配置，建议先导出备份
