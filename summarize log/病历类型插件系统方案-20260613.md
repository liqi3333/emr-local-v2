# 病历类型插件系统方案 — 2026-06-13

## 一、需求背景

### 1.1 原始需求

用户希望在现有 7 个病历类型标签基础上，增加**一级标签**（病历文档种类），形成双层标签结构：

```
一级标签（文档种类）          二级标签（病历类型）
┌─────────────────┐
│ 临床医师病历  ←─── 首次病程录、主治查房、主任查房、术前小结、术前讨论、手术记录、出院小结
│ 同意书       ←─── 手术同意书、输血同意书、麻醉同意书、...
│ 护理记录     ←─── 护理评估、护理计划、护理记录单、...
└─────────────────┘
```

### 1.2 功能范围

- **一级标签**：可配置（动态添加/删除/开关）
- **二级标签**：可配置（动态添加/删除/开关）
- **每个一级标签下的二级标签**：完全独立

### 1.3 初始分类（示例）

| 一级标签 | 二级标签 |
|---------|---------|
| 临床医师病历 | 首次病程录、主治查房、主任查房、术前小结、术前讨论、手术记录、出院小结 |
| 同意书 | 手术同意书、输血同意书、麻醉同意书、... |
| 护理记录 | 护理评估、护理计划、护理记录单、... |

## 二、当前架构分析

### 2.1 命名不一致问题

当前有三套命名系统：

| 位置 | 命名风格 | 示例 |
|------|---------|------|
| 数据库 `type` 列 | 完整名 | `firstCourse`, `attendingRound`, `chiefRound` |
| 提示词模板键 | 简写名 | `emr`, `attending`, `chief` |
| URL 路径 | 简写名 | `/emr/generate`, `/attending/generate` |

### 2.2 硬编码位置统计

| 文件 | 硬编码内容 | 数量 |
|------|-----------|------|
| `database.js` | `RECORD_DATA_COLUMNS` 数组 + switch/case | 7 个 case |
| `store.js` | 状态键名 | 7 个键 |
| `index.html` | 标签按钮 + 预览容器 | 14 个元素 |
| `EmrPreview.js` | LABELS/KEYS 对象 + 渲染方法 | 7 组 |
| `ChatArea.js` | 字段描述 switch/case | 7 个 case |
| `api.js` | 路由端点 | 7 个端点 |
| `ai-mock.js` | Mock 函数 | 7 个函数 |
| `templates.js` | 模板常量 + getter | 7 组 |

### 2.3 数据流

```
HTML tabs (data-tab)
  → EmrPreview._bindTabs() → store.activeTab
  → ChatArea 读取 store.activeTab → 构建系统提示词
  → EmrPreview._regenerate() → api.generate*()
  → POST /api/<type>/generate
  → promptTemplates.assembleSystemPrompt('<templateKey>')
  → AI 返回 JSON → store.state.<storeKey>
  → EmrPreview._renderActiveTab() → 渲染
  → _saveRecord() → db.saveRecord() → database._buildRecordContent()
```

### 7.3 列管理策略

```javascript
// database.js 改动
function getEnabledColumns() {
  const registry = JSON.parse(db.getSetting('record_registry') || '[]');
  const enabledTypes = registry
    .filter(cat => cat.enabled)
    .flatMap(cat => cat.types.filter(t => t.enabled));
  
  return enabledTypes.flatMap(type => 
    type.fields.map(f => f.key)
  );
}
```

### 8.1 类型管理路由

```

PUT    /api/types/registry              → 更新注册表
POST   /api/types/category              → 添加一级标签

PUT    /api/types/category/:id          → 更新一级标签
POST   /api/types/category/:id/type     → 添加二级标签
DELETE /api/types/category/:id/type/:typeId → 删除二级标签
PUT    /api/types/category/:id/type/:typeId → 更新二级标签
```

### 8.2 动态 AI 生成路由

```javascript
// api.js 改动
// 当前：硬编码路由
router.post('/emr/generate', ...);
```

## 九、关键代码改动示例

### 9.1 EmrPreview.js — 动态标签生成

```javascript
// 当前（硬编码）
_bindTabs() {
  const tabBtns = document.querySelectorAll('.pane-tab');
  tabBtns.forEach(btn => {
    btn.addEventListener('click', () => {
      this._activeTab = btn.dataset.tab;
      store.setState({ activeTab: this._activeTab });
      this._renderActiveTab();
    });
  });
}

// 新设计（动态）
_renderCategoryTabs() {
  const container = document.getElementById('categoryTabs');
  container.innerHTML = '';
  
  const registry = getRegistry();
  registry.filter(cat => cat.enabled).forEach(category => {
    const btn = document.createElement('button');
    btn.className = `category-tab ${category.id === this._activeCategory ? 'active' : ''}`;
    btn.dataset.category = category.id;
    btn.textContent = `${category.icon} ${category.label}`;
    btn.addEventListener('click', () => {
      this._activeCategory = category.id;
      store.setState({ activeCategory: category.id });
      this._renderCategoryTabs();
      this._renderTypeTabs();
    });
    container.appendChild(btn);
  });
}

_renderTypeTabs() {
  const container = document.getElementById('typeTabs');
  container.innerHTML = '';
  
  const category = getCategoryById(this._activeCategory);
  category.types.filter(t => t.enabled).forEach(type => {
    const btn = document.createElement('button');
    btn.className = `type-tab ${type.id === this._activeTab ? 'active' : ''}`;
    btn.dataset.tab = type.id;
    btn.textContent = `${type.icon} ${type.label}`;
    btn.addEventListener('click', () => {
      this._activeTab = type.id;
      store.setState({ activeTab: type.id });
      this._renderTypeTabs();
      this._renderActiveTab();
    });
    container.appendChild(btn);
  });
}
```

## 十、风险评估

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 大量硬编码重构 | 高 | 分阶段实施，阶段一只做 UI 双层标签 |
| 数据库 schema 变更 | 中 | 使用 ALTER TABLE 添加列，保持向后兼容 |
| AI 提示词模板缺失 | 中 | 新类型使用基础模板，后续逐步完善 |
| 性能影响 | 低 | 注册表只加载一次，缓存在 Store 中 |
| 现有数据迁移 | 中 | 更新现有记录的 category 字段 |
