## 一、任务目标回顾

在病历系统 v2 中增加一套 GUI 提示词模板编辑功能：

1. 保留系统默认提示词，可随时恢复；
2. 支持创建/另存为自定义提示词模板文件；
3. 可设置当前 AI 生成使用哪套模板；
4. 按 7 个病历标签页组织，每个标签页下按病历输出字段顺序编辑；
5. 不允许手动增删字段，新增字段通过默认模板更新自动同步；
6. 不改动病历主页面（`index.html` 病历标签、`EmrPreview.js`、病历状态等）。

## 七、实施后问题修复

### 问题现象

用户反馈主页出现以下问题：
- 左侧疾病栏完全空白；
- 7 个病历标签页丢失/不响应；
- 各按钮点击无反应。

### 根因

`public/js/services/api.js` 文件末尾在追加模板管理函数时，误留了重复的闭合代码：

```js
export async function getPromptTemplateStatus(name) {
  // ...
  return res.json();
}
  return res.json();   // ← 多余代码
}                      // ← 多余代码
```

浏览器以 ES Module 方式解析 `api.js` 时遇到语法错误，导致模块加载失败。由于 `app.js` 第一行就 `import * as api from './services/api.js'`，整个应用初始化流程中断，所有组件（DiseaseTree、EmrPreview、ChatArea 等）均未实例化。

### 修复

删除 `public/js/services/api.js` 末尾多余的 `return res.json(); }` 两行代码。

### 验证

- `node --check public/js/services/api.js` 通过；
- 服务可正常启动，`/api/health` 返回正常；
- 主页 HTML 结构和各组件容器元素均存在；
- 提示词管理 API 接口可正常响应。

### 二次问题：主页"提示词"按钮点击无反应

**现象**：直接访问 `http://localhost:8000/prompts` 正常，但点击主页 header 上的"✏️ 提示词"按钮无反应。

**根因**：在 `public/index.html` 中添加了按钮，但未在 `public/js/app.js` 中绑定点击事件。

**修复**：在 `public/js/app.js` 中新增 `setupPromptsButton()` 函数，并在 `init()` 中调用：

```js
function setupPromptsButton() {
  const btn = document.getElementById('btnPrompts');
  btn?.addEventListener('click', () => {
    window.open('/prompts', '_blank');
  });
}
```

（使用 `window.open('/prompts', '_blank')` 实现新标签页打开，不离开当前病历页面。）

### 二次验证

- `node --check public/js/app.js` 通过；
- 服务可正常启动；
- `app.js` 中 `setupPromptsButton` 函数已定义并在 `init()` 中被调用。

### 三次问题：提示词编辑器切换模板后仍显示默认模板

**现象**：在 `/prompts` 页面，保存自定义模板后，再次从下拉菜单选择该模板，内容区仍显示默认模板。

**根因**：`PromptEditor.js` 中切换模板时：
1. 仅通过 `<option selected>` 属性设置选中项，部分浏览器在 `innerHTML` 替换后可能未同步 `select.value`；
2. 异步加载新模板期间没有清空旧内容，也没有 loading 反馈，可能残留默认模板视图。

**修复**：在 `public/js/components/PromptEditor.js` 中：

1. `_renderTemplateSelect()` 中设置 `innerHTML` 后，强制同步：
   ```js
   select.value = this._state.currentName;
   ```

2. 下拉框 `change` 事件中，切换时立即清空旧内容并显示 loading：
   ```js
   this.container.querySelector('#peTemplateSelect')?.addEventListener('change', (e) => {
     this._state.currentName = e.target.value;
     this._state.currentTemplate = null;
     this._state.editingField = null;
     this._setLoading(true);
     this._render();
     this._loadCurrentTemplate().then(() => {
       this._setLoading(false);
       this._render();
     });
   });
   ```

- `node --check public/js/components/PromptEditor.js` 通过；
- 服务可正常启动；
- 后端模板 CRUD API 正常；
- 浏览器获取的 `PromptEditor.js` 包含修复代码。
